Green IA

PyTorch · Tensorflow · Deep Learning · StreamLit · CSV · JSONL · Numpy · Pandas · Transformers

Nous avons imaginé Green IA avec deux amis, dans l'objectif d'accompagner les consommateurs dans leur choix de produits alimentaires🌱. Vous le savez peut-être déjà, mais en complément du Nutri-Score a été créé un Éco-score, encore expérimental à ce jour, mais qui progressivement sera ajouté aux emballages de nos produits alimentaires 🛒. Tout comme le Nutri-score, l'Éco-score attribue une note à un produit allant de A (très bon) à E (très mauvais), en fonction de son impact sur l'environnement 🌍 : destruction de la couche d'ozone, impact sur la biodiversité, consommation d'eau, etc. 💧


Notre objectif a donc été de fournir aux utilisateurs une prédiction de cette note pour tous les produits de la base de données OpenFoodFacts, qui est une base de données libre d'accès, notamment utilisée par l'application Yuka 📱.


Nous avons ainsi développé une application web disponible sur ordinateurs et mobiles 💻📱, permettant de scanner le code-barres d'un article, puis d'afficher le résultat de la prédiction 🔍.


Ces prédictions sont réalisées par un modèle hybride fait maison, comprenant deux perceptrons multicouches : un pour les données numériques, et un autre pour traiter les sorties du premier perceptron ainsi que du modèle Transformers Bert qui traite nos données textuelles 📊📚. En sortie, nous obtenons la note de l'article, qui est juste dans 94% des cas ✅. Dans les 6% restants, l'erreur est toujours d'une catégorie inférieure, par exemple, nous avons prévu un B au lieu d'un A 🅱️➡️🅰️.


Ce modèle est inclus dans une pipeline automatique qui télécharge toutes les semaines les données de la base, les traite, puis entraîne notre modèle 🔄. Si ce modèle offre de meilleures performances que le précédent, il le remplace pour réaliser de nouvelles prédictions sur l'ensemble des nouvelles données 📈.
